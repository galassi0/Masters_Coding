This folder contains the MATLAB code used to generate 3D trajectories in cartesion coordinates and linear velocity for the KUKA 500-3 robotic arm.
/n
Formats the output to match the .dat and .src files similar to those generated by RoboDK.

calcPose_Kuka will calculate positional changes, rotating around a fixed radius. Takes initial position as center of radius to rotate around. Position data is a vector in degrees. Outputs values as 1x6 coordinates [x,y,z] position, [z,y,x] orientation (order: zyx for rotations), and tangential velocity.

calcPose_Kuka_diag_v3 is built similarly but for angled trajectories (eg positons 45 degrees off of the anterior-posterior axis).

generateKUKALinearDAT: generates the appropriate .dat file using PTP translation for the initial move, and linear moves for all subsequent movements (consider input position vector's resolution for smooth curves), with matching file names to the .src file. Has the option for an output sync pulse for EMG or IMU synchronization.
generateKUKALinearSRC: generates the matching .src file for the trajectory.

The KUKA pendant will require both files to be uploaded for operation.
